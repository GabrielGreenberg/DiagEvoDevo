## Live decisions / gotchas since last handoff
- **Default mode is now strong/0.2, not weak.** Tests that pin the OLD weak numbers carry an
  explicit `mode: 'weak'` override — if you touch coincidence, keep that split (weak arm
  byte-identical to HEAD; strong arm the honest default). `scripts/probe_promotion_repin.ts`
  is the workflow that recomputes the pins — never guess a repin.
- **Strong = weak × alignment × ink.** The ink gate `g = ‖disp‖²/(‖disp‖²+θ_ink²)` is the
  collapse killer: a segment shrunk to a point has every point-reading path trivially
  coincident, and g = 0 refuses it. This is WHY strong closes the weak version's collapse
  traps (dot plot, mid-anchored bars). Don't "simplify" the gate away.
- **Golden's strong ink factor at the acceptance K is 0.773, not 1.0** — the dataset's
  smallest bars pay θ_ink; loud golden ≈ 0.98. This is honest and documented in tests/UI;
  it is not a bug.
- **Measurement paths** (`core/measurements/paths.ts`): run/rise dogleg convention is
  **parallel leg first** (corner C = (end_x, start_y) in v1 geometry). A vertical bar's rise
  leg IS its ink. Angle pairs and origin-free page-point projections have no linear ink-path
  and keep the weak formula in strong mode (documented, not an oversight).
- **Circular rungs are live** (`0984dca`): the full chain ordinal ≤ interval ≤ ratio ≤ cyclic
  is restored with SOUND forms. Interval-on-angles = Mardia circular–linear R² (`fIntCirc`,
  computed sqrt-free); ratio-on-angles = the unchanged v2.1 `fRatio`. `fIntCircExact` was
  ε-guarded this arc after an antipodal-cancellation defect (R² > 1 at exactly-antipodal
  bearings) — keep the ε on the sqrt-free Mardia det. NO reading is structurally blocked from
  any relation now (user directive) — only the manual Readings toggles exclude.
- **Coincidence weight is a basin-selection dial, not a truth dial** (like data-ink's 0.5).
  Weak at 0.3 stabilized two collapse traps in full-depth optimization; 0.2 restores 6/6
  legible. Strong closes the traps structurally, but 0.2 stays the certified weight.
- **The --quick 1200-step cap misses legibility even at w=0** — a cap artifact, not a

## Standing principles — reproduced verbatim from CLAUDE.md, every handoff (do not paraphrase, do not drop)

### I. Depth over patches
Add every feature at the deepest correct layer. If a change belongs in the
architecture, change the architecture — refactor rather than bolt on. A surgical
patch that avoids a warranted structural change is a defect, not a shortcut. When
unsure whether a change is deep or shallow, choose deep, even at the cost of a larger
refactor now. We are building a conceptual instrument; its internal structure must
stay faithful to the ideas in `CONCEPT.md`, not accumulate convenience hacks.

### II. Workflows over re-derivation
Build and verify through repeatable workflows — scripts, task runners, test suites —
not by re-reading files or re-deriving results inside the context window. Prefer
invoking a workflow to reconstructing knowledge. This holds for **construction** as
much as for verification: if you find yourself doing the same multi-step thing twice,
make it a workflow first. This preserves context length and makes every result
reproducible on demand.

### Working mode
Autonomous build–verify–iterate: design into the architecture → implement → write
adversarial tests that try to break it → run the test workflow → iterate to green → record
in PROGRESS.md. Verification is the approval; don't wait for sign-off on self-checkable steps.
